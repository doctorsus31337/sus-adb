# Instrumentation & Root Readiness Advisor

This official add-on classifies the explicitly selected device into an
evidence-based Frida route. It covers existing rooted-server readiness,
operator-supplied server setup, Gadget preparation through APK Laboratory,
debuggable and emulator routes, ADB-only inspection, and blocked or unknown
states.

The host owns device selection, ADB and Frida services, local file selection,
assessment-scope checks, worker execution, and every state-changing
confirmation. Server setup actions remain separate: select, validate, preview,
upload, set executable, start, configure managed forwarding, verify, stop, and
remove only a server managed by the current SUS Companion session.

The add-on also parses read-only boot observations and safely inspects bounded
operator-selected firmware metadata. It never downloads firmware, acquires or
hides root, exploits, unlocks, patches boot images, flashes, wipes, changes
verified boot, installs persistence, or automatically patches or installs an
APK.

Bootloader unlocking commonly wipes user data and must not be used as a
recovery technique.
