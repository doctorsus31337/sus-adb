# Step-by-step tutorial

1. Export or copy this directory to a developer location. 2. Choose a unique stable lowercase ID and never reuse the official `susadb.skeleton-module` ID. 3. Change the display name and update the semantic version. 4. Keep API version `1.0`. 5. Review every capability, platform, entry point, contribution, and `addon_ui.ui_mode` (`embedded`, `window`, or `hybrid`). 6. Implement immutable models and structured results. 7. Add bounded cancellation-aware workers for long work. 8. Request host callback marshalling rather than retaining Tk. 9. Test with fakes. 10. Install the completed derivative through Plugin Manager and explicitly review its new digest. The exported copy is editable and independent of bundled source; it is not installed, trusted, enabled, loaded, or executed automatically. The host owns windows, singleton behavior, theme, geometry, closing, and cleanup.
