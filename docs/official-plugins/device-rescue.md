# Device Rescue & Recovery

Use only existing authorized ADB/recovery/root access and explicitly selected sources/destination/limits. Shared-storage presets never include private app data. Private paths require existing root, active storage and sensitive-data scope, exact preview, and confirmation. Duplicate policy is explicit; files are hashed and manifested. No lock/FRP/decryption bypass, root acquisition, unlock, flash, wipe, device deletion, upload, or persistence exists. Unlocking commonly wipes data and is not a recovery step.
