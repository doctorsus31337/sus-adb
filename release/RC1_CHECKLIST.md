# SUS-ADB 1.0 RC1 checklist

Automated checks cover unit tests, compilation, startup self-test, resource verification, migrations, and privacy audit. The following representative-machine checks remain manual and must not be marked complete until performed.

- [ ] Clean clone and dependency installation
- [ ] Linux packaged launch
- [ ] Windows packaged launch
- [ ] No-device launch
- [ ] Physical-device selection
- [ ] Instrumentation and Script Studio
- [ ] Pentest case creation
- [ ] ADB, Runtime, Network, Storage, APK Lab, Findings/Reports, and Plugins
- [ ] Shutdown cleanup
- [ ] Package and checksum verification
- [ ] Privacy audit
- [ ] Upgrade and configuration migration
